Core surfaces: Sequence Builder, Sequence Library, Pose Library, Public Sequence, Public Profile + Schedule, Admin.

The challenge

Instructors still plan classes in notes, spreadsheets, or memory — fine until you need reusable pose libraries, variations, and sequences you can actually share. Newer instructors also need a way to rehearse cues with a real clock and live transitional announcements, not a phone timer beside a written list. Sculpt Architect had to feel like a craft tool and still work as multi-user SaaS.

My approach

I owned information architecture, builder UX, auth, permissions, public surfaces, and the backend path — including migrating persistence from Supabase to Turso/libSQL. The builder carries sections, timed poses, and nested group blocks so instructors can plan a real class — not just a flat list.

What shipped

  • Sequence Builder — sections, pose instances, group blocks, drag reorder, duration calculation, templates / clone
  • Pose Library — 50+ poses and variations as first-class objects
  • Sequence Library — browse and deep-link into saved work
  • Public surfaces — shareable sequence URLs and public profiles with schedule editing
  • Auth — magic-link verification plus credential login
  • Admin console — user search, ban/unban, 5 permission toggles
  • Assets — Vercel Blob uploads for poses and profiles

How the system works

React 18 + Vite + React Router front end. Hono API with JWT and bcrypt. Turso / libSQL data. Hosted on Vercel (sculpt-sequences). Role-flag permissions so instructor and admin tooling share one product.

How the product works

Sculpt Architect is the loop instructors run every week: sign in, build a timed sequence, reuse poses from a library, share publicly when ready, and keep multi-instructor access under admin control.

Surface 01

Auth into a private workspace

Goal: get instructors into a private workspace

Magic-link verification and credential login drop instructors into their own practice workspace — sequences stay private until they choose to publish. That boundary mattered before any of the craft tools were useful.

Surface 02

Teach on a timer

Goal: teach on a timer (play / pause / next)

The sequence runner is what turns a plan into a class: play, pause, and advance while the clock runs. Instructors stay on the floor with students instead of flipping between a spreadsheet and a phone timer.

Sequence runnerView repo

Surface 03

Builder → library → public share

Goal: close the craft loop from draft to deep link

Sequences start in the builder (sections, pose instances, group blocks, drag reorder, duration). Saved work lands in the Sequence Library; poses and variations live as first-class objects in the Pose Library (50+ catalog). When an instructor is ready, public sequence URLs and profile pages carry the work outside the account — a deep link should deepen into a profile and schedule, not dead-end.

Surface 04

Admin & permissions as ops

Goal: multi-instructor orgs without spreadsheet access control

Admin search, ban/unban, and five permission scopes keep instructor and admin tooling in one product. That was ops readiness for studios and multi-instructor orgs — who can edit libraries, who can publish, who gets cut off — before the user base outgrew a single owner account.

Role flags on JWT-backed accounts: five toggles plus ban controls so access doesn’t live in a side spreadsheet.

Reflection

What mattered was shipping a complete loop — auth, library, share, and admin — so instructors could run real classes in one product, not a pile of half-finished screens.
